# Example: Yoga AI Voice Companion

This is a complete real-world example of documents generated by `init-project` for an AI-powered Iyengar yoga voice companion.

## Project Profile

| Attribute | Value |
|-----------|-------|
| Type | Full-stack web app |
| Stack | Next.js 14, React 18, TypeScript, Tailwind CSS, Prisma, PostgreSQL |
| Key integrations | Claude Agent SDK, Volcengine RTC (voice), Knowledge Base API |
| Language | Chinese (UI, docs, commits) / English (code) |
| Stage | Active development (MVP) |

## Generated Documents

| File | Lines | Description |
|------|-------|-------------|
| [CLAUDE.md](CLAUDE.md) | 84 | Agent onboarding — stack, structure, commands, conventions |
| [DESIGN.md](DESIGN.md) | 231 | Full design system — sage green palette, Noto fonts, breathing-rhythm motion |
| [docs/requirements.md](docs/requirements.md) | ~120 | 7 functional modules (M-001 to M-007) with acceptance criteria |
| [docs/architecture.md](docs/architecture.md) | ~150 | 4-layer architecture with ASCII diagrams and decision quick-ref |
| [docs/decisions.md](docs/decisions.md) | ~90 | 7 decisions (D001-D007): RTC, LLM choice, framework, DB, design |
| [progress.md](progress.md) | ~50 | Module status with emoji tracking |

## What Makes This Example Good

1. **CLAUDE.md** is 84 lines — dense, no filler, every line serves the Agent
2. **DESIGN.md** has concrete values for every token (hex, px, bezier curves)
3. **decisions.md** records rejected alternatives, not just chosen options
4. **Cross-referencing works**: D001-D007 in decisions.md match architecture.md's quick-ref table
5. **Module numbering** (M-001 to M-007) is consistent across requirements.md, architecture.md, and progress.md
6. **Out of Scope** section explicitly blocks feature creep (no video recognition, no multi-user)
