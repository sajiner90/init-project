## 写作指南

**ASCII 图的价值：**
- 比文字描述直观 10 倍
- 不需要外部工具渲染
- 在终端、GitHub、编辑器中都能正常显示
- 画不好没关系，信息准确比美观重要

**按项目类型调整侧重：**
| 项目类型 | 重点 section | 可以简化/省略 |
|---------|-------------|-------------|
| 前端 | 页面路由、状态管理、组件层级 | 数据模型（如用 localStorage） |
| 后端 | 服务分层、数据模型、中间件链 | UI 相关 |
| 全栈 | 前后端边界、数据流 | 都需要 |
| CLI/库 | 模块划分、公共 API surface | 页面路由、状态管理 |

**技术决策速查表的作用：**
- 快速回答"为什么用 X 不用 Y"
- 编号对应 decisions.md 中的详细记录
- Agent 在做技术选型时应先查此表

**模块的"关键文件"：**
- 已有项目：必须指向真实存在的文件路径
- 新项目：写出计划的文件路径，标注"(planned)"
