## 写作指南

**模块编号规则：**
- M-001 起始，递增分配，永不复用
- 编号与 progress.md、architecture.md 保持一致
- 新增模块追加到末尾，不重排

**优先级定义：**
| 级别 | 含义 | 标准 |
|------|------|------|
| P0 | Must have | 没有它产品不可用 |
| P1 | Should have | 第一次用户测试前需要 |
| P2 | Nice to have | QA/迭代阶段补充 |

**验收标准写法：**
- ✅ 好："用户提交表单后 2s 内看到成功提示"
- ❌ 差："表单提交体验流畅"
- ✅ 好："体式过滤后至少保留 3 个体式，否则显示提示"
- ❌ 差："过滤逻辑正确"

**Out of Scope 的价值：**
- 防止 Agent 在 session 中擅自添加功能
- 防止需求蔓延（scope creep）
- 明确说明"为什么不做"比只说"不做"更有价值

## 真实案例片段

```markdown
### M-003: 练习引导 (P0)
**描述：** AI Agent 根据编排序列，通过语音实时引导用户完成体式练习。包含体式指令播报、保持时长计时、过渡提示。
**验收标准：**
- [ ] AC-1: Agent 按序列逐一播报体式名称和进入指令
- [ ] AC-2: 每个体式保持时长结束时有声音/视觉提示
- [ ] AC-3: 用户可通过语音说"跳过"切换到下一个体式
- [ ] AC-4: 练习全流程不超过 200 轮对话（maxTurns 安全阀）
- [ ] AC-5: Agent 在最后一个体式完成后引导进入挺尸式

### 暂不实现（Out of Scope）
- 多用户同时练习 — MVP 仅支持单用户
- 视频动作识别 — 需要摄像头权限，隐私风险高
- 自定义编排 — 用户不具备编排能力，由 AI 决定
```
